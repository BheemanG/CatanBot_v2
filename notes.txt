WS OUT (sequence always contains incrementing out_sequence):
    Action 0 (Send chat message)
        payload: message string
    Action 2 (Dice Roll)
        payload: true
    Action 3 (Place Robber)
        payload: tile #
    Action 6 (End Turn)
        payload: True
    Action 7 (Confirm Discard selected)
        Payload: cards being discarded
    Action 8 (Update discard list)
        payload: card #s for discarded (1, 1, 1) = 3 lumber
    Action 9 (Buy dev card)
        payload: true
    Action 10 (highlight road select at bottom is valid) kinda useless, can just build road directly on map with action 11?
        payload:true
    Action 11 (Place Road after initial settlement)
        payload: single number, represetnting the edge? (ex: 71)
    Action 12 (Build Road)
        payload: edge road was built
    Action 14 (highlight settlement at bottom - useless)
        payload:true
    Action 15 (Place Initial Settlement)
        payload: vertex # (ex: 48)
    Action 16 (build settlement)
        payload: vertex # of settlement
    Action 17 (click on buy city at bottom)
        payload: true
    Action 18 (confirm build city)
        payload: city vertex
    Action 19 (build city)
        payload: settlement vertex
    Action 21 (occurs right after road building dev card action 11)
        payload: edge you built road
    Action 47 (writing trade or counteroffer) / cancel highlighted option / cancel dev (don't have to cancel, can just take other action directly)
        payload: true
    Action 48 (confirm dev card that I highlighted)
        payload: 12 for vp (idk why), 11 for robber, 14 for road building
    Action 49 (send trade/counteroffer)
        see json for details
        CounterOfferInResponseToTradeId: null if not a counteroffer
    Action 50 (respond to trade)
        id: trade id when came in
        response: number (0 = accept, 1 = decline, 2 = creating counter offer, 3 = canceled counter offer?)
    Action 51 (finalize trade after enemy agrees)
        see json
    Action 53 (highlight a dev card at bottom)
        payload: 48 (48 for all of them idk why)
    Action 54 (Pause Game)
        payload:true
    Action 64 (win game? end game?)
        payload: true
    Action 66 [USELESS] (hover/select to place a road or settlement off initial settlement) - first placement is sequence = 2
        payload: single number, represetnting the edge or vertex (ex: 71) or null if canceling
        immediately sends null ws_out to cancel self and then does action 15 or 11 after confirmation
    Action 67 (sends an error message when switching between manual and ws inputs)
        payload: bug id
